Guia para iniciantes

Instalar o Git na sua máquina

Pesquise sobre como instalar o git no sistema operacional que você usa. Um tutorial no YouTube pode ajudar muito. Mas em geral, você só precisa seguir o passo a passo que já vem no site do Git:

Clique aqui para acessar a página de download do Git!

Trabalhando em seu projeto com o Git

Agora você já pode começar a fazer o controle das versões do projeto usando Git. A primeira coisa a ser feita é entrar no repositório do projeto e "copiar" os arquivos pra sua máquina.

Copie o endereço do repositório

Em seguida entre no seu terminal e escreva

git clone https://github.com/SeuNomeDeUsuario/SeuRepositorio.git

Depois de feito isso, haverá uma pasta no seu computador com os arquivos do repositório. Para ter certeza disso, escreva "ls"no seu terminal. Esse comando vai listar todos os arquivos no diretório em que você está.


Entre na pasta do repositório pelo terminal

cd NomeDoRepositorio

Se o nome for muito comprido, você pode escrever as primeiras letras e apertar TAB, e o terminal completa pra você.

Agora você pode fazer as alterações, e não precisa de um editor de código pra isso, qualquer alteração é válida, mesmo se for feita no bloco de notas.

Usando o VS Code


Você não precisa obrigatoriamente de um editor de código. Pode editar o documento no bloco de notas ou no próprio terminal usando o VIM, mas editores de código facilitam a escrita.

Procure sobre como instalar o VS Code no seu sistema operacional

Com ele instalado, você pode criar arquivos para seu projeto

Crie um de arquivo de teste pelo próprio VS Code, e nomeie ele de "teste.html".

Escreva "html" no seu arquivo e você verá que o VS Code te sugere algumas opções de autocomplete. Selecione o HTML 5.

Agora você tem a estrutura completa de um HTML.

Escreva:

<h1> Olá, mundo! </h1>

No canto esquerdo do seu VS Code, onde está seu arquivo HTML, clique nele com o botão direito do mouse e selecione "Copy Path"

Abra seu navegador e cole o path no espaço da URL.

Você deve estar vendo seu site

Porém, essa visualização não atualiza automaticamente toda vez que você mexe documento.

Para isso, precisamos adicionar uma extensão ao seu editor: o Live Server





Na parte mais à direita do VS Code, vá na opção Extensions, pesquise por Live Server e instale.

Para usá-lo, no canto inferior direito do VS Code, vai ter a opção "Go Live". Assim que você clicar nela, será aberta uma aba do seu navegador com seu site renderizado!

Enviando suas alterações para o repositório remoto

Você já fez as alterações no documento e agora vai enviar para o repositório no Github. Para isso, volte ao terminal, na pasta do projeto, e dê os seguintes comandos:

git status
git add .
git commit -m "seu comentario aqui"
git push

Se não lembra como entrar na pasta do projeto, lembre-se de digitar "ls" no seu terminal. Assim você vai saber em qual diretório está.

"Diretório" é um nome pra "pasta". Se estiver na pasta errada, pode voltar digitando "cd .." (dois pontinhos mesmo) no terminal, e "ls" pra ver se está na pasta certa.

Novamente: quando a pasta do projeto aparecer, entre nela digitando:

cd NomeDoRepositorio

Comandos Git e terminal

clone clona os arquivos do repositório
status mostra as condições dos arquivos do seu diretório
add leva as alterações que você fez pra área de staging
commit salva as mudanças no repositório
push atualiza o repositório remoto com as mudanças feitas

Algumas dicas


Aprenda a pesquisar

Grande parte da rotina de um desenvolvedor consiste em buscar uma solução pro seu problema. Diferentemente de outras áreas, você não precisa gravar comandos, termos, fórmulas etc.

O principal é aprender a lógica, e as outras coisas acompanham. Algumas coisas você vai acabar decorando pela repetição, mas até lá abuse do Google. Saber como pesquisar é muito importante.

Eventualmente você vai tentar executar um comando no terminal e receber uma mensagem de erro. Leia essa mensagem, o terminal te conta exatamente o que houve de errado. Caso você não entenda, pesquise sobre aquele problema! Você não precisa resolver tudo sozinho, algumas coisas não são triviais, mas dá pra resolver sim bastante coisa com curiosidade e iniciativa.

Boas práticas Git

Quando você fizer uma alteração no documento a ser commitado, comente sobre a alteração feita, se atendo aos seguintes aspectos:

Especifique a alteração:
Todos os membros da equipe devem saber do que se trata aquele commit, então evite comentários vagos e opte por termos mais precisos sobre o que você fez. Por exemplo, se você fez uma alteração no estilo de todos os <h1>, escreva "altera estilo h1"; se você exclui um link da barra de navegação, comente "exclui nomeDoLink da navegação". E por aí vai.
Não escreva muito:
Seu commit deve ser breve, não faça muitas descrições sobre a sua alteração. O ideal é que você coloque um verbo no imperativo + o objeto afetado.
Não faça commit se não tiver tudo certo:
Evite commitar os arquivos se você fez apenas uma parte da alteração e ainda está desenvolvendo, ou sem antes checar se está tudo ok com o seu trabalho.
Não termine o commit com um ponto final
